A Restricted Boltzmann Machine in R

 Posted on March 17, 2021  |  5 minutes

github source for this project

I’ve taken an interest in a succession of binary network models: the hopfield network, Boltzmann Machine, the Restricted Boltzmann Machine (RBM), and the Helmholtz Machine. There isn’t much implementation of these in R, but I did find this tutorial. Each model in some way learns the joint probability distribution of many binary variables using binary hidden nodes.

The difference between the RBM and the Helmholtz machine is the former uses a kind of Gibbs sampling of the hidden states to compute gradients of node weights, whereas the ladder is a generative network that learns by “dreaming” fake data during the sleep phase of the wake-sleep algorithm. The solution of a Helmholtz Machine is supposed to minimize the Kullbeck-Liebler divergence of the real data and the generative distribution. In a way that I don’t fully understand yet, this leads to learning deeper hidden structure that better matches the data-generating structure. I found through this project that the boltzmann machine with more than one layer is more likely than not to fall into a local solution that looks nothing like the data-generating network.

My goal was to learn about both models by coding them from scratch in R. Then I could test the difference for myself. As the example code that I found was not abstracted to take any model dimensions (number of layers, nodes per each layer), I had to think of a way to do this that would allow quick and convenient manipulations. I decided to set up a list object L containing the name and location of each part of the model:

> L
[[1]]
[1] "bias"

[[2]]
 [1] "x1"  "x2"  "x3"  "x4"  "x5"  "x6"  "x7"  "x8"  "x9"  "x10" "x11" "x12" "x13" "x14" "x15" "x16" "x17" "x18" "x19" "x20"

[[3]]
[1] "h1_1" "h1_2" "h1_3" "h1_4" "h1_5" "h1_6" "h1_7" "h1_8"

The top level indices (1-3) are the layers, and the strings within each are the names of nodes. This way, I could set up the network as one big adjacency matrix with contents of L as dimnames (inspired by the RAM notation of structural equation models). Then, I could code the entire thing with simple matrix multiplications while using L to index block sub-matrices representing the connections between layers. This approach is slow because it is in R and it involves a lot of unnecessary computations over all the zero-values, but it seemed like a balance between making the code abstract and concise.

Starting with just the RBM for now, this code obtains the gradient by generating random node values up the network conditional on lower layers, then back down to the data conditional on the upper layers.

calcGrad <- function(X, Wr, Wg) {
  Gr <- Gr2 <- Wr
  for (i in 3:Q) {
    X[, L[[i]]] <- (X %*% t(Wr))[, L[[i]]]
    X[, L[[i]]] <-  matrix(rbinom(prod(dim(X[, L[[i]]])), size = 1, c(lgt(X[, L[[i]]]))), dim(X[, L[[i]]])[1], dim(X[, L[[i]]])[2])
    Gr[L[[i]], unlist(L[c(1, i - 1)])] <- (t(X) %*% X / nrow(X))[L[[i]], unlist(L[c(1, i - 1)])]
  }
  Gr[L[[2]], L[[1]]] <- (t(X) %*% X / nrow(X))[L[[2]], L[[1]]]
  X2 <- X
  for (i in (Q - 1):2) {
    X2[, L[[i]]] <- (X2 %*% t(Wg))[, L[[i]]]
    X2[, L[[i]]] <-  matrix(rbinom(prod(dim(X2[, L[[i]]])), size = 1, c(lgt(X2[, L[[i]]]))), dim(X2[, L[[i]]])[1], dim(X2[, L[[i]]])[2])
  }
  for (i in 3:Q) {
    X2[, L[[i]]] <-  lgt( (X2 %*% t(Wr))[, L[[i]] ] ) 
    if(i<Q) X2[, L[[i]]] <- matrix(rbinom(prod(dim(X2[, L[[i]]])), size = 1, c( X2[, L[[i]]] )), dim(X2[, L[[i]]])[1], dim(X2[, L[[i]]])[2])
    Gr2[L[[i]], unlist(L[c(1, i - 1)])] <- (t(X2) %*% X2 / nrow(X))[L[[i]], unlist(L[c(1, i - 1)])]
  }
  Gr2[L[[2]], L[[1]]] <- (t(X2) %*% X2 / nrow(X) )[L[[2]], L[[1]]]
  dG <- Gr - Gr2
  return(dG)
}

I regret that R lists subscript with double brackets [[]] so they kind of dominate the code. I guess the rest of it is pretty ugly too.
